# DSA Logs & Algorithmic Encyclopedia

A comprehensive, highly structured repository documenting my journey through competitive programming and advanced Data Structures and Algorithms (DSA). 

This repository serves as both a personal log of solved problems and an algorithmic encyclopedia, containing optimal C++ implementations for canonical problems across major competitive programming platforms.

## 🗂️ Repository Structure

The repository is organized by topic to mirror a professional competitive programming roadmap. Each directory contains solutions ranging from brute-force approaches to highly optimized algorithms.

### Core Data Structures & Algorithms
- **`Arrays_and_Hashing/`**: Frequency maps, prefix sums, and array manipulation.
- **`Two_Pointers/`** & **`Sliding_Window/`**: Optimal subarray searches and window management.
- **`Binary_Search/`**: Standard search, rotated arrays, and binary search on answer.
- **`Stacks_and_Queues/`**: Monotonic stacks, queues, and sliding window maximums.
- **`Trees/`**: Traversals, LCA, and BST validation.

### Advanced Topics
- **`Graphs/` & `Advanced_Graphs/`**: DFS/BFS, Dijkstra's, MST (Prim/Kruskal), Max Flow (Dinic), and 2-SAT.
- **`Dynamic_Programming/`**: Memoization, tabulation, and state optimization.
- **`Greedy/`**: Local optimization strategies.
- **`Recursion_and_Backtracking/`**: State space exploration and pruning.
- **`Heaps_and_Priority_Queues/`**: Top-K elements and median tracking.
- **`Tries/`**: Prefix trees and XOR tries.
- **`String_Algorithms/`**: KMP, Manacher's, and Aho-Corasick.
- **`Advanced_Data_Structures/`**: Segment Trees, Binary Indexed Trees (Fenwick), and Sparse Tables.
- **`Advanced_Math/` & `Advanced_Geometry/`**: Euler's Totient, Chinese Remainder Theorem, Sweep Line algorithms, and Shoelace formula.

## 💻 Platforms Covered
Solutions are sourced from a variety of platforms, emphasizing different problem-solving aspects:
- **LeetCode**: Interview-style algorithmic challenges.
- **Codeforces**: Heavy mathematical and logic-based competitive programming.
- **CSES Problem Set**: Classic algorithm implementations and data structure design.

## ⚙️ Implementation Details
All solutions are written in modern **C++**. I strive to include:
1. **Time and Space Complexity Analysis** for every approach.
2. **Multiple Approaches**: Often starting with a naive Brute Force and iterating towards the Optimal solution.
3. **Clean Code**: Adhering to good naming conventions and competitive programming best practices.
